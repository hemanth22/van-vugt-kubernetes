Kubernetes is all about cloud-native computing. In cloud-native computing, application-specific information is not stored on a server, but in cloud. This means that common configuration, like configuration files and startup parameters need to be stored in cloud also, and that is what the ConfigMap is used for. In this lab you'll learn how to use a ConfigMap to provide environment variables to your application.
 
 In the following steps you will learn the following:
 
*	Create a ConfigMap
*	Update the application to use the ConfigMap
*	Analyze how the application is using the ConfigMap
